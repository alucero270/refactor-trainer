# Bad Exercise Examples

## Example 1 - Gives Away the Solution

Bad exercise:

> Extract three helper functions called `validate_input`, `transform_data`, and `save_result`, then move the loop into `transform_data`.

Why it is bad:

- reveals the final structure
- turns the exercise into implementation instructions
- removes the learner's design work

## Example 2 - Too Broad

Bad exercise:

> Clean up this file, improve naming, reduce duplication, reorganize responsibilities, and make it more maintainable.

Why it is bad:

- combines multiple major refactors
- gives no primary issue
- creates vague success criteria

## Example 3 - Cosmetic Only

Bad exercise:

> Reformat this function and make the spacing more consistent.

Why it is bad:

- focuses on cosmetic churn
- does not teach refactoring judgment
- conflicts with the anti-rules

## Example 4 - Hidden Final Answer in Hint Form

Bad exercise:

> Refactor this function by introducing guard clauses at the top, extracting a helper for logging, and moving the API call into a repository object.

Why it is bad:

- leaks too much structural guidance up front
- collapses the hinting ladder into the exercise itself
- narrows the solution space too aggressively
