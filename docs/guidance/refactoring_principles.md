# Refactoring Principles

Refactor Trainer defines "better code" for MVP using the following principles:

- Prefer readability over cleverness.
- Reduce cognitive load for future readers.
- Improve naming clarity so intent is obvious.
- Reduce responsibility overlap.
- Minimize change risk by localizing changes.
- Improve testability and isolation.
- Prefer maintainability over stylistic purity.

## Anti-Rules

The guidance system must not:

- flag purely stylistic differences as refactoring targets
- reward formatting-only churn
- treat subjective preferences as objective problems

## Usage

These principles should guide:

- exercise generation
- hint generation
- difficulty interpretation
- acceptance review of generated content
